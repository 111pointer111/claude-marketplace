# 历史人物 Persona 蒸馏流水线

> 本文件为 OpenClaw 每日定时任务的执行规则。
> 目标：从原始史料自动蒸馏历史人物 persona，输出标准化 SKILL.md，上传至 GitHub 仓库。
> 状态管理：所有执行状态记录在 `DONE.md` 中，OpenClaw 自主读取并决定下一个处理人物。

---

## 一、流水线入口

### 触发机制

- **执行频率：** 每日北京时间 06:00
- **触发条件：** DONE.md 中「下一待处理」有有效条目，且该人物未在 done/ 中
- **并发限制：** 每次仅处理 1 个人物

### 入口操作（每次执行前必做）

```
1. 读取 DONE.md 中的「下一待处理」节
2. 确认人物名称
3. 读取 backlog.md 中该人物的完整条目（含 sources 列表）
4. 检查 done/{姓名}.done 是否存在 → 存在则跳过此人，更新「下一待处理」为下一个 priority high 人物
5. 若 backlog.md 中所有 priority high 人物均已完成，重新计算下一个最高优先级人物
6. 若 backlog.md 中所有人均已完成 → 流水线终止，记录"DONE.md 全量完成"
```

### 人物选择规则

```
优先级顺序（由高到低）：
1. priority: high，在 backlog.md 中排在最前
2. priority: medium，在 backlog.md 中排在最前
3. priority: low，在 backlog.md 中排在最前

已出现在 DONE.md「已完成」表格中的人 → 永不重复处理
pipeline 失败的人 → 保留在 backlog.md 中，在 DONE.md「执行日志」记录 failed，等待人工介入
```

## 二、Stage 1：资料采集（含分类回退链）

**目标：** 获取人物的原始文本语料。

**执行步骤：**

```
1.1 读取 backlog.md 中该人物的 sources 列表

1.2 对每个 source URL 按类型抓取：

    IF type == "poetry_collection" OR type == "collected_works" OR type == "personal_essay":
        → 使用 WebFetch 抓取全文原始文本
        → 保存到 raw/{人物名}/{类型}_{序号}.txt
        → 标注每段/每首诗的写作年份（如可考证）

    IF type == "historical_record":
        → 抓取传记正文，保存到 raw/{人物名}/传记.txt
        → 提取其中直接引语，单独保存到 raw/{人物名}/引语.txt

    IF type == "commentary":
        → 抓取后人评述，保存到 raw/{人物名}/后世评述.txt

1.3 分类回退链补充（根据人物类型选择对应策略）

    先按以下规则确定人物类型：
    - chinese_poet：中国诗人/文学家（备注含"诗人/词人/诗词/文学"）
    - chinese_philosopher：中国哲学家（备注含"儒家/道家/理学/哲家"）
    - chinese_scientist：中国科技人物（备注含"科技/医学/数学/工程"）
    - chinese_emperor：中国帝王将相（备注含"帝王/皇帝/军事/宰相"）
    - chinese_artist：中国艺术家（备注含"书法/绘画"）
    - western_philosopher：西方哲学家（朝代含"外国/古希腊/德国/法国"）
    - western_writer：西方文学人物（朝代含"外国" + 文学相关）
    - western_scientist：西方科学家（朝代含"外国" + 科学相关）

    【中国诗人】baike.baidu.com 传记 → gushiwen.cn 诗词 → SerpAPI 补充 → 降级
    【中国哲学家】baike.baidu.com 传记 → 核心著作原文 → SEP(如适用) → 降级
    【中国科学家/帝王/艺术家】baike.baidu.com 传记 → Britannica/WHE 补充 → 无需诗词
    【西方哲学家】SEP → Britannica → WHE → 百度百科(如有) → 无需诗词
    【西方文学人物】WHE → Britannica → SerpAPI 补充 → 百度百科(如有) → 无需诗词
    【西方科学家】Britannica → SEP(如有) → WHE → 无需诗词

    每类源只需成功抓取 1-2 个即可停止。成功标准：WebFetch 返回有效正文。

1.4 语料完整性检查
    → 是否有传记？→ 否 → 搜索"{人物名} 生平"补充
    → 全部语料是否 ≥ 3000 字？→ 否 → 标记 WARN
    → 是否有直接引语？→ 否 → 搜索"{人物名} 名言"补充
    → 写作年份标注：尽可能标注，无法确定的标注"年份未详"

1.5 合并：全部语料 → raw/{人物名}/全部语料.txt
    （引语.txt 和 后世评述.txt 单独保留，不合并）
```

**输出目录结构：**
```
raw/
└── 苏轼/
    ├── 诗词_1.txt           # 含年份标注
    ├── 文集.txt
    ├── 东坡志林.txt
    ├── 传记.txt
    ├── 引语.txt              # 直接引语
    └── 后世评述.txt
```

**质量标准：**
- 诗词语料 ≥ 50 首（不足 30 首则 confidence 上限为 medium；哲学家/科学家/帝王类人物不受此限）
- 文集/散文语料 ≥ 20 篇（非诗人不受此限）
- 有传记依据
- 全部语料合并为 raw/{人物名}/全部语料.txt 供后续使用

## 三、Stage 2：阶段切分

**目标：** 将人物一生划分为若干阶段。

**执行步骤：**

```
2.1 读取 raw/{人物名}/传记.txt 和诗词年份标注
2.2 将一生划分为 2-5 个阶段，划分依据：
    ├── 重大人生转折点（贬谪/升迁/战乱/亡国等）
    ├── 创作风格明显变化的节点
    └── 年龄节点（通常：早年 0-30，中年 30-55，晚年 55+）

2.3 对每个阶段输出：
    - 起止年份
    - 主要生活地点
    - 该阶段核心事件（3-5条）
    - 代表作品列表
    - 该阶段语言风格概述
    - confidence: high（现存作品 ≥ 30篇）/ medium（10-29篇）/ low（< 10篇）

2.4 输出文件：processed/{人物名}/stages.md
```

## 四、Stage 3：多维蒸馏

---

### 4.1 思想内核（跨时期不变的核心）

```
分析指令：
你是一位古典文学与思想史专家。请从以下语料中提炼出"{人物名}"的思想内核。
思想内核是这个人在任何时期、任何作品中都保持不变的核心价值观与底线。

在输出前请先思考以下深度问题，融入最终结论：

1. 时代坐标系：此人与同时代其他人相比最大的思想差异是什么？
2. 言行一致性：此人的主张与实际行为是否一致？有矛盾则判断"时代局限"还是"人格矛盾"
3. 内核 vs 权重：哪些是真正贯穿一生的核心，哪些只是特定时期的侧重？
4. 内部张力：是否存在内在矛盾或张力？这是弱点还是思想深度的来源？

输出 JSON，要求每条结论至少附 2 处原文引用，必须可从 raw/ 原始文本回溯。

{
  "核心价值观": [
    {
      "条目": "...",
      "描述": "...",
      "原文引用": ["..."],
      "同时代对比": "...",
      "贯穿程度": "始终如一 / 有演变 / 晚期淡化",
      "实践一致性": "一致 / 部分矛盾 / 明显矛盾"
    }
  ],
  "政治立场": {"描述": "...", "原文引用": [...], "时代坐标": "...", "内在张力": "...", "confidence": "high/medium/low"},
  "底线清单": [{"条目": "...", "描述": "...", "原文引用": "...", "是否有例外": "有/无"}],
  "自我定位": {"描述": "...", "原文引用": [...], "与时代关系": "..."},
  "思想内部张力": {"主要矛盾": "...", "深层原因": "...", "对人格的影响": "..."}
}
```

**输出：** `processed/{人物名}/dimension_思想内核.json`

### 4.2 语言特征（量化评分 + 深度分析）

```
分析指令：
你是一位古典文学风格分析专家。请分析{人物名}的作品，提取语言特征。
目标是让对话 Agent 能准确模仿此人的语言风格。

分析范围：诗词 {N} 首 + 文 {M} 篇

请提取以下 11 个维度，每维度给出评分和原文佐证：

1. 文言/白话光谱（1=全文言，5=大量白话入诗）
2. 雅/俗光谱（1=极典雅，5=俚语民谣风格）
3. 骈偶 vs 散行（骈%估算）
4. 句长偏好：短句(≤7字)% / 中句(8-15字)% / 长句(≥16字)%
5. 语气标签（可多选）：慷慨/沉郁/清淡/诙谐/悲凉/豪迈/婉约/冷峻/炽热/隐忍
6. 高频意象 Top 10（含出现次数估算和例句）
7. 修辞偏好（可多选）：用典/白描/比喻/对偶/夸张/反问/借代/比兴
8. 文体偏好排序：诗/词/文/赋/书信/奏议
9. 负面特征：此人刻意回避什么表达？什么修辞几乎不存在？
10. 节奏韵律特点：句读节奏、骈散转换感、诵读时的听觉感受
11. 同时代独特性：与同类型作者相比最大特色是什么？用一个比喻形容其风格

输出格式：JSON
```

**输出：** `processed/{人物名}/dimension_语言特征.json`

---

### 4.3 表达偏好（对外表达策略 + 通用特征）

```
分析指令：
你是一位修辞学专家。请分析"{人物名}"对不同对象的表达策略差异。

请分析以下方面，并增加通用表达特征：

1. 对**上位者**（君王/权贵）→ 直谏/婉谏/讽刺/沉默？附原文
2. 对**友人**（平辈/知己）→ 戏谑/真挚/克制？附原文
3. 对**敌人/政敌** → 犀利/绵里藏针/回避？附原文
4. 对**百姓/民生** → 平实/悲悯/激愤？附原文
5. 对**自己**（内心独白/自嘲）→ 风格？附原文

通用表达特征（新增）：
6. 幽默/讽刺程度：几乎不用 / 偶尔 / 频繁 / 标志性风格
7. 情感外放程度：极度内敛 / 温和克制 / 坦率直接 / 热烈奔放
8. 调节能力：在不同身份/亲密程度的人面前语言调节幅度
9. 禁忌话题处理：沉默回避 / 转移话题 / 正面拒绝 / 委婉推脱

无直接语料时，标注"[无直接语料，据诗文推测]"。
输出格式：JSON
```

**输出：** `processed/{人物名}/dimension_表达偏好.json`

### 4.4 立场光谱

```
分析指令：
请为"{人物名}"在以下议题上打分。
评分：-2（极端反对）到 +2（极端支持），0=中立/无关

议题：皇权专制 / 变法革新 / 民生疾苦 / 华夷之辨 / 文学功利观 / 出世/入世

重要规则：
1. 每项评分必须附原文依据
2. 如某议题完全无涉及，标注"语料不足"，设 0 并说明
3. 每项评分标注置信度 high/medium/low
4. 如立场随时间演变，标注演变轨迹（如"早期+2，晚期-1"）

输出 JSON：
{
  "皇权专制": {"评分": ±2, "置信度": "high/medium/low", "说明": "...", "依据": "...", "演变": "..."},
  "变法革新": {"评分": ±2, "置信度": "...", "说明": "...", "依据": "...", "演变": "..."},
  ...
  "综合可信度": "high/medium/low",
  "综合说明": "..."
}
```

**输入：** raw/{人物名}/全部语料.txt + raw/{人物名}/后世评述.txt
**输出：** `processed/{人物名}/dimension_立场光谱.json`

### 4.5 阶段一致性校验

```
分析指令：
请对比"{人物名}"不同人生阶段的语言风格与思想倾向：

1. 哪些特征在所有阶段保持一致？（核心风格不变的部分）
2. 哪些特征随阶段明显变化？（发展的部分）
3. 思想内核在所有阶段是否保持一致？
   如有矛盾，判断：a)"史料误差" 或 b)"人物真实思想发展"，附判断依据
4. 语料来源可信度评估：
   - 一手史料（直接原文/语录）占比高/中/低？
   - 二手文献（传记/评述/研究）占比高/中/低？
   - 核心结论是否在多个来源中交叉验证？
   - 是否存在被LLM过度解读或虚构的风险？
5. 每个阶段的 confidence 评级是否合理？

输出：一致性报告（markdown）+ 语料可信度评估 + 最终 confidence 调整建议
```

**输出：** `processed/{人物名}/consistency_check.md`

## 七、质量控制

### 自动校验规则

```
规则1: 引用可回溯
→ SKILL.md 中每一处引用，必须能在 raw/{人物名}/ 原始文本中找到对应原文
→ 任何引用无法回溯 → 标记 WARN，输出时注明"[引用来源待核实]"

规则2: 语言特征与原文一致
→ 维度评分(1-5)与佐证文本是否匹配？
→ 不一致 → 标记 WARN，人工复核

规则3: 置信度守门
→ 诗词 < 30 首 → overall_confidence ≤ medium
→ 诗词 < 10 首 → overall_confidence = low
→ 某阶段作品 < 10 篇 → 该阶段 confidence = low

规则4: 成语/典故验证
→ SKILL.md 中出现的成语典故，执行一次 WebSearch 验证来源
→ 错引 → 标记 WARN 并修正

规则5: 自我矛盾处置
→ 若人物在不同时期存在立场矛盾：
    a) 优先判断为"史料误差" → 标注并排除
    b) 确为"人物真实发展" → 如实呈现，标注"反映思想发展"
    c) 不强制统一，保留复杂性
```

### 输出前强制检查清单

```
□ 所有引用均可从 raw/ 目录回溯
□ 每条维度结论至少有 1 处原文引用支撑
□ overall_confidence 符合统计规则
□ 立场光谱中每个议题至少有 1 处原文依据
□ SKILL.md 正文（不含引用）字数在 1500-5000 字之间
□ done/{人物名}.done 已创建
□ GitHub 已 push 成功
□ DONE.md 已更新（追加完成记录 + 更新下一待处理 + 更新统计）
□ backlog.md 中该人物条目已移除
```

## 八、GitHub 上传与 DONE.md 更新

### 执行顺序

```
8.1 生成 output/{人物名}/ 所有文件
8.2 Git 操作
    → git add output/{人物名}/
    → git add done/{人物名}.done
    → git commit -m "distill: {人物名} persona ({confidence})"
    → git push origin main

8.3 Git push 成功后，更新 DONE.md：
    a) 在「已完成」表格追加一行
    b) 更新「统计概览」
    c) 从 backlog.md 中移除该人物条目
    d) 读取 backlog.md 下一个最高优先级人物
    e) 更新「下一待处理」节

8.4 错误处理
    → 若 git push 失败，保留本地缓存，次日重试
    → 重试 3 次仍失败 → 在 DONE.md「执行日志」中记录失败，
      在 DONE.md「下一待处理」中标注 failed，等待人工介入
    → 若 Git push 成功但 DONE.md 更新失败 → 单独提交 DONE.md 更新
```

## 九、人物队列（backlog.md 格式）

> backlog.md 仅保留 pending 状态的人物，已完成的人自动移除。
> DONE.md 是唯一状态记录。

```markdown
| 人物 | 朝代 | priority | 备注 |
|------|------|----------|------|
| 苏轼 | 北宋 | high | 全才，史料最丰富 |
| 杜甫 | 唐 | high | 分期风格变化显著 |
| ... | ... | ... | |
```

**Priority 定义：**
- high：史料极丰富（诗词100+首，文集完整，传记详尽），可生成完整版
- medium：史料较丰富（诗词30-100首），可生成标准版
- low：史料有限（诗词<30首），生成精简版
- failed：执行失败，需人工介入
