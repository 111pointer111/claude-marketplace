# 历史人物 Persona 蒸馏流水线

> 本文件为 OpenClaw 每日定时任务的执行规则。
> 目标：从原始史料自动蒸馏历史人物 persona，输出标准化 SKILL.md，上传至 GitHub 仓库。

---

## 一、流水线入口

### 输入格式

流水线接收一个任务文件 `queue/today.md`，每日凌晨由调度器更新，内容格式：

```markdown
## 2026-04-17

### 今日人物

- name: 苏轼
  dynasty: 北宋
  lifespan: "1037-1101"
  priority: high
  sources:
    - url: "https://ctext.org/shiji/304.zh"
      type: historical_record
      description: "《宋史·苏轼传》"
    - url: "https://zh.wikisource.org/wiki/東坡樂府"
      type: poetry_collection
      description: "东坡乐府全集"
    - url: "https://zh.wikisource.org/wiki/东坡志林"
      type: personal_essay
      description: "东坡志林"
    - url: "https://ctext.org/dongpo.txt.zh"
      type: collected_works
      description: "苏轼文集"
```

### 调度规则

- **执行时间：** 每日北京时间 06:00
- **前置检查：** 若 `queue/today.md` 不存在或人物已在 `output/` 中完成（`done/`），跳过
- **并发限制：** 每次仅处理 1 个人物，避免 API 调用过载

---

## 二、流水线阶段（Step by Step）

---

### Stage 1：资料采集

**目标：** 获取人物的原始文本语料。

**执行步骤：**

```
1.1 读取 queue/today.md 中的 sources 列表
1.2 对每个 source URL 执行以下操作：

    IF type == "poetry_collection" OR type == "collected_works" OR type == "personal_essay":
        → 使用 WebFetch 抓取全文原始文本
        → 保存到 raw/{人物名}/{类型}_{序号}.txt
        → 标注每段/每首诗的写作年份（如可考证）

    IF type == "historical_record":
        → 抓取传记正文
        → 保存到 raw/{人物名}/传记.txt
        → 提取其中直接引语（用引号标记），单独保存

    IF type == "commentary":
        → 抓取后人评述
        → 保存到 raw/{人物名}/后世评述.txt
        → 用于 Step 3.4 的外部视角校验

1.3 补充搜索（如果原始 sources 不够丰富）：
    → 执行 WebSearch: "{人物名} 诗词全集", "{人物名} 文集", "{人物名} 书信"
    → 从搜索结果中补充 1-2 个权威来源
    → 保存到 raw/{人物名}/补充_{序号}.txt
```

**输出目录结构：**
```
raw/
└── 苏轼/
    ├── 诗词_{序号}.txt       # 含年份标注
    ├── 文集.txt
    ├── 东坡志林.txt
    ├── 传记.txt
    ├── 引语.txt              # 从传记中提取的直接引语
    └── 后世评述.txt
```

**质量标准：**
- 诗词语料 ≥ 50 首（不足则标注 confidence: low）
- 文集/散文语料 ≥ 20 篇
- 有传记依据

---

### Stage 2：分龄切片切分

**目标：** 将人物一生划分为若干阶段，每个阶段单独建模。

**执行步骤：**

```
2.1 读取 raw/{人物名}/传记.txt 和年谱资料
2.2 根据人物生平，将一生划分为 3-5 个阶段：

    划分依据：
    ├── 重大人生转折点（贬谪/升迁/战乱/亡国等）
    ├── 创作风格明显变化的节点
    └── 年龄节点（通常：早年 0-30，中年 30-55，晚年 55+）

2.3 对每个阶段，提取：
    - 起止年份
    - 主要生活地点
    - 该阶段核心事件（3-5条）
    - 代表作品列表
    - 该阶段语言风格概述
    - confidence: high/medium/low（由该阶段现存作品数量决定）

2.4 输出阶段切片清单：
    ```
    ## 阶段切片

    ### 阶段一：早年（眉山/汴京）1037-1065
    confidence: medium
    - 代表作：《和子由渑池怀旧》《送安惇秀才失解西归》
    - 风格：意气风发，锋芒外露，少沉淀

    ### 阶段二：中年巅峰（密州/徐州/湖州）1069-1086
    confidence: high
    - 代表作：《念奴娇·赤壁怀古》《江城子·密州出猎》《水调歌头》
    - 风格：豪放与细腻并存，技艺圆熟

    ...以此类推
    ```
```

**输出文件：** `processed/{人物名}/stages.md`

---

### Stage 3：多维度蒸馏

对每个切片，按以下 5 个维度逐一提取。

---

#### 3.1 思想内核（跨时期不变的核心）

**执行命令：** LLM 分析 + 原文引用支撑

```
分析指令：
你是一位古典文学与思想史专家。请从以下语料中提炼出"{人物名}"的思想内核。
思想内核是这个人在任何时期、任何作品中都保持不变的核心价值观与底线。

请输出：
1. 核心价值观（1-3条，每条附原文引用 2-3 处）
2. 政治/社会立场（对君/民/敌/改革的态度）
3. 底线清单（什么是此人绝对不妥协的，用原文标注）
4. 自我定位（此人如何看待自己/自己的使命）

输出格式：JSON
{
  "核心价值观": [{"条目": "...", "原文引用": [...]}],
  "政治立场": {...},
  "底线清单": [...],
  "自我定位": "..."
}
```

**输入：** raw/{人物名}/全部语料.txt（诗词+文集+传记）
**输出：** `processed/{人物名}/dimension_思想内核.json`

---

#### 3.2 语言特征（最可自动化的维度）

**执行命令：** LLM 量化分析（最成熟的步骤，可高度模板化）

```
分析指令：
你是一位古典文学风格分析专家。请分析以下{人物名}的作品，提取语言特征。

分析范围：
- 诗词 {N} 首
- 文 {M} 篇
- 书信 {K} 封（如有）

请提取以下维度，每维度给出评分（1-5）和原文佐证：

1. **文言/白话光谱**（1=全文言，5=大量白话入诗）
2. **雅/俗光谱**（1=极典雅，5=俚语民谣风格）
3. **句式偏好**：骈偶 vs 散行（骈% / 散%）
4. **句长偏好**：短句（5字内）% / 中句 / 长句（20字+）%
5. **语气标签**（可多选）：慷慨/沉郁/清淡/诙谐/悲凉/豪迈/婉约/冷峻/炽热/隐忍
6. **高频意象**（Top 10）：列出最常出现的自然/人文意象
7. **修辞偏好**（可多选）：用典/白描/比喻/对偶/夸张/反问/借代/比兴
8. **文体偏好排序**：诗/词/文/赋/书信/奏议（按使用频率排序）

输出格式：JSON（便于程序处理）
```

**输入：** raw/{人物名}/诗词_{序号}.txt + raw/{人物名}/文集.txt
**输出：** `processed/{人物名}/dimension_语言特征.json`

---

#### 3.3 表达偏好（对外表达策略）

**执行命令：** 专项分析

```
分析指令：
你是一位修辞学专家。请分析"{人物名}"对不同对象的表达策略差异。

请分析以下语料中，该人物：
1. 对**上位者**（君王/权贵）→ 措辞是直谏/婉谏/讽刺/沉默？
2. 对**友人**（平辈/知己）→ 措辞是戏谑/真挚/克制？
3. 对**敌人/政敌** → 措辞是犀利/绵里藏针/回避？
4. 对**百姓/民生** → 措辞是平实/悲悯/激愤？
5. 对**自己**（内心独白/自嘲）→ 措辞风格？

每条结论附原文依据。

输出格式：JSON
```

**输入：**
- raw/{人物名}/引语.txt（对上位者/敌人的话）
- raw/{人物名}/书信（如有，对友人/知己的话）
- raw/{人物名}/诗词（如有对民生/自我的表达）

**输出：** `processed/{人物名}/dimension_表达偏好.json`

---

#### 3.4 立场光谱（价值观判断）

**执行命令：** 立场标注

```
分析指令：
你是一位政治哲学与伦理专家。请为"{人物名}"在以下争议性问题上的立场打分。

评分标准：-2（极端反对）到 +2（极端支持），0=中立/无关

议题维度：
- 皇权专制：对此人如何看待绝对皇权？
- 变法革新：支持改革还是守旧？（王安石变法期间此人是何立场？）
- 民生疾苦：是否关心底层？
- 华夷之辨：民族立场如何？
- 文学功利观：文以载道 vs 文学独立？
- 出世/入世：道家隐逸倾向 vs 儒家济世倾向？

每条附原文依据。
输出：JSON立场向量
```

**输入：** raw/{人物名}/全部语料.txt + raw/{人物名}/后世评述.txt（交叉验证）
**输出：** `processed/{人物名}/dimension_立场光谱.json`

---

#### 3.5 阶段一致性校验

**执行命令：** 跨阶段对比

```
分析指令：
请对比"{人物名}"不同人生阶段的语言风格与思想倾向，回答：

1. 哪些语言特征在所有阶段保持一致？（核心风格不变的部分）
2. 哪些语言特征随阶段明显变化？（发展的部分）
3. 思想内核（维度3.1）在所有阶段是否保持一致？
   如有矛盾，判断是"史料误差"还是"人物真实发展"，给出判断依据
4. 每个阶段的 confidence 评级是否合理？

输出格式：
- 一致性报告：markdown
- 最终 confidence 调整建议：{阶段名: "high/medium/low", 理由: "..."}
```

**输入：** processed/{人物名}/dimension_语言特征.json（各阶段）+ dimension_思想内核.json
**输出：** `processed/{人物名}/consistency_check.md`

---

### Stage 4： Persona Card 生成

**目标：** 将所有维度整合为标准化 SKILL.md 格式。

**执行步骤：**

```
4.1 读取 processed/{人物名}/ 所有维度文件
4.2 按模板生成 SKILL.md（格式见 Stage 5）
4.3 生成 README.md（使用说明 + 人物简介）
4.4 生成 METADATA.json（元数据：蒸馏日期、版本、置信度）
4.5 生成 CITATIONS.md（所有原文引用，注明出处，供人工校验）
```

**输出文件：**
```
output/{人物名}/
├── SKILL.md           # 核心 persona 文件
├── README.md          # 使用说明
├── METADATA.json     # 元数据
├── CITATIONS.md      # 原文引用清单
└── raw_stats.json    # 原始数据统计（处理了多少诗词/文）
```

## 三、质量控制

### 自动校验规则

```
规则1: 语言特征一致性
→ 每个维度的评分（1-5）与原文佐证是否匹配？
→ LLM 自评不一致时，输出警告，人工复核

规则2: 成语/典故准确性
→ SKILL.md 中所有引用的成语和典故，在输出前执行一次验证：
→ WebSearch "{人物名} {成语/典故}" 确认来源准确
→ 错引率 > 5% 则输出 WARN 并暂停提交

规则3: 引用原文一致性
→ 检查所有"原文引用"是否与 raw/{人物名}/ 原始文本一致
→ 严禁 LLM 自行发挥，引用必须可回溯

规则4: 置信度守门
→ 某阶段作品数 < 10篇 → confidence 强制为 low
→ 某阶段无直接引语留存 → 表达偏好中"对XX"标注为"无直接语料，推测"
```

### 输出前的强制检查清单

```
□ 所有引用均可从 raw/ 目录中的原始文本回溯
□ 每条维度结论至少有 1 处原文引用支撑
□ 成语/典故已通过搜索验证
□ confidence 评级符合统计规则
□ 立场光谱中每个议题至少有 1 处原文依据
□ 不存在同一阶段内明显矛盾的描述
□ 字数：SKILL.md 正文（不含引用）在 1500-5000 字之间
```

## 四、GitHub 上传流程

### 目录结构

```
skills/
├── README.md                    # 全量人物索引（每次上传后自动更新）
└── {人物名_pinyin}/
    ├── SKILL.md
    ├── README.md
    ├── METADATA.json
    ├── CITATIONS.md
    └── raw_stats.json

done/                            # 已完成任务记录
└── {日期}_{人物名}.done          # 空文件，标记完成
```

### 上传执行步骤

```
4.1 生成 diff
    → 检查 output/{人物名}/ 中是否有新文件或变更

4.2 更新 skills/README.md 索引
    → 在索引末尾追加新人物条目
    → 格式："| {人物名} | {朝代} | {定位} | {confidence} |"

4.3 Git 操作
    → git add output/{人物名}/
    → git add skills/README.md
    → git add done/{日期}_{人物名}.done
    → git commit -m "distill: {人物名} persona ({confidence})"
    → git push origin main

4.4 错误处理
    → 若 push 失败（网络/权限），保留本地缓存，次日重试
    → 最多重试 3 次，超过则告警并跳过
```

## 五人物队列管理

### 队列格式（queue/daily.md）

```markdown
## 队列总览

| 人物 | 朝代 | priority | status |
|------|------|----------|--------|
| 苏轼 | 北宋 | high | pending |
| 杜甫 | 唐 | high | pending |
| 陶渊明 | 晋 | medium | pending |
| ... | ... | ... | ... |

## 已完成
（执行后自动移至此）

## 执行日志
- 2026-04-17: 苏轼 ✅ (confidence: high)
- 2026-04-18: 杜甫 ⏳ (执行中)
```

### 优先级规则

```
priority: high
→ 史料极丰富（诗词100+首，文集完整，传记详尽）
→ 至少 3 个可考证的人生阶段
→ 预期生成完整版 persona（所有维度均有充足语料）

priority: medium
→ 史料较丰富（诗词30-100首，主要文集留存）
→ 1-2 个明确的人生阶段
→ 生成标准版 persona

priority: low
→ 史料有限（诗词<30首，大量散佚）
→ 仅能建模部分维度
→ 生成精简版 persona（仅核心切片）
```

## 七、关键约束与限制

1. **只蒸馏有充分史料支撑的人物**。不凭空捏造语言风格，史料不足则 confidence 降级。
2. **引用必须回溯原文**。SKILL.md 中每一处引用，必须能在 raw/ 目录中找到原始文本。
3. **不进行价值观现代化解读**。忠实还原人物在其时代语境下的立场，不以后世价值观评判。
4. **每天最多完成 1 个人物**。质量优先，不追求数量。
5. **置信度强制标注**。输出必须包含 overall_confidence，不允许模糊处理。
6. **自我矛盾的处置规则**：若发现人物在不同时期或不同文本中存在立场矛盾：
   - 优先判断为"史料误差"（引用伪造/误植）→ 标注并排除
   - 确为"人物真实矛盾" → 如实呈现，标注"该矛盾可能反映人物的思想发展"
   - 不强制统一，保留复杂性
