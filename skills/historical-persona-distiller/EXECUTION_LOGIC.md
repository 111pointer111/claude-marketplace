> 本文件是流水线的核心逻辑。OpenClaw 按此文件的步骤逐一执行。
> 所有 LLM prompt 均以「真实执行」为标准编写，可直接复制使用。

---

## 预检查

```
## 预检查 — 每次执行前必做

1. 读取 DONE.md，找到「下一待处理」中的人物名（记为 TARGET）
2. 检查 done/{TARGET}.done 是否存在：
   - 存在 → 跳过此人，重新从 backlog.md 取下一个 priority high pending 人物
   - 不存在 → 继续执行
3. 从 backlog.md 读取 TARGET 的完整条目：
   - priority
   - dynasty
   - lifespan
   - sources 列表（每个含 url / type / description）
4. 记录开始时间
```

## Stage 1 — 资料采集

### Step 1.1：抓取原始文本

```markdown
## Stage 1.1 — 资料采集

对 backlog.md 中 TARGET 的每个 source URL，执行：

如果 type == "poetry_collection" 或 "collected_works" 或 "personal_essay"：
    → 使用 WebFetch 抓取 URL 全文
    → 保存到 raw/{TARGET}/{type}_{序号}.txt
    → 标注每首诗/每段文字的写作年份（如有）

如果 type == "historical_record"：
    → 使用 WebFetch 抓取传记正文
    → 保存到 raw/{TARGET}/传记.txt
    → 从正文中提取所有用引号包围的"直接引语"，保存到 raw/{TARGET}/引语.txt

如果 type == "commentary"：
    → 使用 WebFetch 抓取
    → 保存到 raw/{TARGET}/后世评述.txt

抓取完成后：
    → 统计诗词总数量
    → 如果诗词 < 30 首，执行补充搜索：
        WebSearch: "{TARGET} 诗词全集 site:ctext.org OR site:zh.wikisource.org"
        WebSearch: "{TARGET} 文集 site:ctext.org"
        从搜索结果中取前 2 个有效 URL，WebFetch 抓取并保存
    → 最后，将所有 txt 文件内容合并为一个文件：
        raw/{TARGET}/全部语料.txt
        （引语.txt 和后世评述.txt 单独保留）
```

### Step 1.2：质量初判

```markdown
## Stage 1.2 — 质量初判

统计 raw/{TARGET}/ 目录下各类文件数量：

诗词数量 → N
文集数量 → M
传记 → 有/无
直接引语数量 → K

判断：
  - N >= 50：诗词 confidence 上限 = high
  - N >= 30 且 N < 50：诗词 confidence 上限 = medium
  - N < 30：诗词 confidence 上限 = low

输出 raw_stats.json：
{
  "人物": "{TARGET}",
  "朝代": "{dynasty}",
  "诗词数量": N,
  "文集数量": M,
  "传记": "有/无",
  "直接引语数量": K,
  "confidence_上限": "high/medium/low",
  "抓取时间": "{ISO时间}",
  "sources_used": [url1, url2, ...]
}
```

### 检查3：置信度守门
规则：
- 诗词总数 < 30 → overall_confidence ≤ medium
- 诗词总数 < 10 → overall_confidence = low

### 检查4：字数检查
- SKILL.md 正文（不含 frontmatter，不含引用块）字数：
  → < 1000 字 → 输出 WARN：数据可能不足
  → 1000-5000 字 → 通过
  → > 5000 字 → 检查是否堆砌，酌情精简

### 检查5：一致性交叉检查
→ 对比 dimension_立场光谱.json 和 stages.md 中的描述
→ 立场光谱中的描述是否与各阶段的风格概述一致？
→ 矛盾 → 标记 WARN，优先信任立场光谱的量化评分

### 检查6：Git push

→ git add done/{TARGET}.done
